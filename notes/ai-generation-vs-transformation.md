# AI 技術選定: 「生成」か「変形」か — IP-Adapter 失敗から学んだこと

キャラクターの表情バリエーションを作るとき、
「画像生成 AI で作る」と「既存画像を変形する」では結果が全く違います。
ep2.0〜ep2.1 の試行錯誤から得た知見をまとめます。

## やりたかったこと

1枚のキャラクター画像から「驚き」「笑顔」「考え中」などの表情バリエーションを複数枚作る。

## 最初のアプローチ: IP-Adapter（生成）→ 失敗

### やったこと

IP-Adapter（Stable Diffusion の画像参照モデル）を使い、
「このキャラクターで、驚いた表情で」という条件で画像を生成した。

```python
# IP-Adapter でのアプローチ（概念コード）
pipeline = StableDiffusionPipeline(...)
pipeline.set_ip_adapter(ip_adapter_model)
output = pipeline(
    prompt="surprised expression",
    ip_adapter_image=character_image,
)
```

### 結果

- 顔の輪郭・目の形・肌の色が微妙に変わる
- 同じキャラクターに「見えない」バリエーションが生成される
- CPU のみ環境では 1枚あたり 20〜30分かかる（実用不可）

### なぜ失敗したか

IP-Adapter は「参照画像のスタイルを保ちながら新しい画像を生成する」ツール。
「同一キャラクターを維持する」ためのツールではない。

生成系の AI は必ず「ゆらぎ」がある。そのゆらぎがキャラクターの一貫性を壊す。

## 次のアプローチ: THA3（変形）→ 成功

### やったこと

THA3（Talking Head Anime 3）は1枚の顔画像を「変形」するモデル。
新しい画像を生成するのではなく、既存画像のピクセルを動かす。

```python
# THA3 でのアプローチ（概念コード）
poser = Poser()  # THA3 の変形エンジン
pose = torch.tensor([joy_params])  # 感情パラメータ
output = poser.pose(character_image, pose)
```

### 結果

- 元の画像から「変形」するため顔の特徴が完全に保たれる
- CPU のみでも数秒で処理完了
- 感情パラメータをスライダーで調整できる（再現性がある）

## 「生成」と「変形」の違いを整理する

| 観点 | 生成（IP-Adapter など） | 変形（THA3 など） |
| --- | --- | --- |
| 仕組み | ノイズから新しい画像を合成 | 既存画像のピクセルを動かす |
| キャラクター一貫性 | 保証できない（ゆらぎあり） | 保証される（元画像が起点） |
| 処理速度（CPU） | 数十分/枚 | 数秒/枚 |
| 向いている用途 | 新キャラクター・背景の生成 | 同一キャラクターの表情変換 |
| 向いていない用途 | 同一キャラクターの再現 | まったく新しい画像の生成 |

## 問いを変えると技術選定が変わる

- ❌「AI で表情バリエーションを **生成** したい」→ IP-Adapter を試す
- ✅「同じキャラクターの表情を **変化させたい**」→ THA3 / FOMM / MediaPipe を使う

「何を作りたいか」ではなく「**どういう変換をしたいか**（生成なのか変形なのか）」を
最初に言語化することで、適切なツールの選択肢が絞れる。

## 教訓

**生成系の AI は「同一性を保ちながら変える」ことが苦手。**
キャラクターのアイデンティティ（顔の特徴・色調・スタイル）を厳密に守りたいなら、
「変形」アプローチが安全。

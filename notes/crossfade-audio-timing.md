# MoviePy crossfade で音声がずれる問題と解決策

ep3.5.1 でシーン間フェードトランジションを実装したときに発見したバグの記録。

## 問題

`concatenate_videoclips(method="compose", padding=-tr_sec)` でクロスフェードを実装したところ、
シーンが進むにつれて音声がどんどん前にずれていく現象が起きた。

```
期待: シーンA（音）→ シーンB（音）→ シーンC（音）
実際: シーンAの音が鳴っている途中でシーンBの音が始まる
     シーンBの音が鳴っている途中でシーンCの音が始まる（累積）
```

2シーン目では 0.5 秒ずれ、3シーン目では 1.0 秒ずれる（`tr_sec=0.5` の場合）。
シーン数が増えるほど後半の音声ズレが大きくなる。

## 原因

`padding=-tr_sec` は映像の**開始タイミング**を前にずらすだけで、音声の duration は変えない。

```
クリップA:  映像 |────────────────|  (2.5秒)
            音声 |────────────────|  (2.5秒)
                                 ^ ここで音声が終わる

クリップB を 0.5秒前から開始:
            映像         |────────────────|
            音声         |────────────────|
                         ^ クリップAの音声末尾 0.5秒に重なる
```

MoviePy は `concatenate_videoclips` で音声を単純に加算合成する。
前のシーンの音声が終わる前に次のシーンの音声が始まるため、両方が同時に鳴ってしまう。
これがシーンを重ねるたびに累積する。

## 解決策

**各クリップの duration を `audio.duration + tr_sec` に伸ばし、音声末尾に無音バッファを作る。**

```python
# 音声は元の duration 分だけ鳴らす（例: 2.0秒）
tone = AudioArrayClip(wave_array, fps=44100)  # duration=2.0

# CompositeAudioClip でラップして duration を明示的に伸ばす
# ラップしないと with_duration() が音声を繰り返してしまう
audio_with_buffer = CompositeAudioClip([tone]).with_duration(2.0 + 0.5)  # 2.5秒

# 映像も同じ duration に伸ばす（フェード用の余白として使われる）
video = ColorClip(size=size, color=color, duration=2.0 + 0.5)

clip = video.with_audio(audio_with_buffer)
```

こうすると、映像が 0.5 秒重なっても音声末尾はすでに無音になっており、
次のシーンの音声と干渉しない。

```
修正後:
クリップA:  映像 |──────────────────|  (2.5秒)
            音声 |─────────────|.....|  (2.0秒音 + 0.5秒無音)

クリップB を 0.5秒前から開始:
            映像         |──────────────────|
            音声         |─────────────|.....|
                                       ^ Aの音声はここで終わっている → 重ならない
```

## 結果

音声ズレが完全に解消した。
3シーン・5シーン・10シーンと増やしても累積ズレは発生しない。

## 教訓

**crossfade は映像だけを考えて実装すると音声が壊れる。**

MoviePy の crossfade は映像レイヤーと音声レイヤーを独立して扱う。
`padding=-tr_sec` が映像の重ね合わせに使われる一方で、音声は duration をそのまま維持する。
「映像を重ねる」と「音声を重ねない」を両立させるには、クリップ側で音声 duration に余白を持たせるしかない。

`CompositeAudioClip([audio]).with_duration(...)` のラップが必要なのも注意点。
`audio.with_duration(...)` だけだと音声が末尾まで繰り返し再生されてしまう。
`CompositeAudioClip` でラップすることで末尾の余白を無音として扱えるようになる。

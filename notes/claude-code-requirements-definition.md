# Claude Code に要件定義をさせる設計パターン

ep4.0 で実践した「AIに要件定義を書かせる」アプローチの記録。

## ポイント: FP への確認事項を AI に自動列挙させる

単に「要件を書いて」と頼むのではなく、**計算モデルの前提として FP に確認が必要な事項を別セクションにまとめてください** という指示を加えると、AI が自分で「ここはドメイン専門家に確認しないと決められない」箇所を列挙する。

これが ep4.0 の核心：AI が自分の限界を認識して、専門家への問いを生成する。

## 実際に使ったプロンプト

```
FPが面談で使うライフイベント家計シミュレーターの要件を整理して、
docs/requirements.md に保存してください。

概要：ライフイベント（住宅購入・出産・教育・老後など）のタイミングと収入を入力すると、
年ごとのキャッシュフロー一覧をExcelで出力する。FPが操作する業務ツール。

計算モデルの前提（インフレ率・運用・年金計算方法など）は
FPに確認が必要な事項として別途まとめてください。
```

## なぜうまく機能したか

- 「業務ツール」「FP が操作する」という文脈を与えたことで、ドメイン知識が必要な箇所が浮かび上がった
- 「確認が必要な事項として別途まとめる」という構造を明示したことで、AI が判断できない部分を自己申告する形になった
- CLAUDE.md に「計算モデルの判断はユーザーに確認を取る」と書いておいたことも影響している可能性がある

## 教訓

AI への依頼は「何を作るか」だけでなく「誰に確認が必要か」を明示すると、
出力の品質と透明性が上がる。

# ディレクティブのスコープ設計

## 問題

台本のディレクティブ（`<!-- zoom: in -->`・`<!-- telop: hidden -->` など）は
「どのクリップまで影響するか」を明確に定義しないと、意図しないクリップに演出が漏れる。

具体的な失敗例（zoom ディレクティブ実装当初）:

```markdown
## シーン5

<!-- speaker: yumu -->
ここはズームなし。

<!-- zoom: in scale=1.3 -->
これが zoom in です。           ← ゆむだけズームしたい

<!-- speaker: sabacyan -->
あ、本当だ。                    ← さばきゃんにもズームが漏れていた（バグ）
```

root cause: zoom をセクション（`##`）単位で管理していたため、セクション内の全クリップに適用された。

## 解決策: per-clip での追跡

ディレクティブをファーストパスでアイテムリストに積み、
セカンドパスでクリップ生成時に「その時点での active な値」を参照する方式に変更。

```python
# ファーストパス: ディレクティブをアイテムとして記録
items = [
    {"type": "narration", "text": "ここはズームなし。"},
    {"type": "zoom",      "value": ZoomConfig(direction="in", scale=1.3)},
    {"type": "narration", "text": "これが zoom in です。"},
    {"type": "speaker",   "value": "sabacyan"},
    {"type": "narration", "text": "あ、本当だ。"},
]

# セカンドパス: active_zoom を追跡しながらクリップを生成
active_zoom = None
for item in items:
    if item["type"] == "zoom":
        active_zoom = item["value"]
    elif item["type"] in ("speaker", "expression"):
        active_zoom = None   # ← 話者・表情変更でリセット
    elif item["type"] == "narration":
        build_clip(..., zoom=active_zoom)  # その時点の zoom を適用
```

## スコープのルール

zoom と telop hidden は同じスコープルールに従う：

| イベント | zoom | telop |
| -------- | ---- | ----- |
| `##` セクション切り替え | リセット | リセット |
| `<!-- speaker: -->` 変更 | **リセット** | リセットしない |
| `<!-- expression: -->` 変更 | **リセット** | リセットしない |
| `<!-- zoom: -->` / `<!-- telop: -->` 記述 | 更新 | 更新 |

zoom は「このクリップを話す人にだけ適用する」という意味合いが強いため、
話者・表情の変更でリセットすることで直感的な動作になる。

## 設計の原則

**「どこからどこまで有効か」を宣言的に制御する。**

ディレクティブのスコープは台本を読む人（制作者）が直感的に理解できる単位で切る。

- `##` 切り替えは「シーンの終わり」であり、演出はリセットされて当然
- 話者変更は「別の人が話し始める」ため、前の人への演出は引き継がない
- 表情変更は「演者が演技を変えた」ため、カメラの演出も更新されるのが自然

この原則を守ることで「書いた場所から次の ## まで有効」という一貫したメンタルモデルが成立する。

## 教訓

- ディレクティブを実装するとき、スコープを最初に定義する。「どこで始まり、どこで終わるか」
- セクション単位での管理は実装が単純だが、セクション内で話者が複数いる台本に対応できない
- per-clip 追跡はファーストパス・セカンドパスに分けることで実装できる
- スコープが直感的でないと台本書きがデバッグに時間を取られる。制作者目線でテストする

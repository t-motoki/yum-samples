# ショートコメディの間（ポーズ）設計

## 「とりあえずポーズを入れる」は間違い

ポーズは「何秒入れればいい」という問題ではない。**この間は何のためにあるか**を答えられなければ置かない。

目的なしのポーズは間延びになる。短すぎるポーズは笑いを殺す。

## 間の種類と目安

| 間の目的 | 秒数の目安 | 例 |
|---|---|---|
| 笑いが発生する場所（オチ着地後） | 1.5〜2.5s | 「なんで触ったの。」の後 |
| 驚き・感情転換後の余韻 | 1.0〜2.0s | 「生成が終わりました。」→ reveal 前 |
| 視聴者に映像を見せる | 3.0〜5.0s | スクリーン録画 PiP |
| エンディングの余韻 | 3.0〜5.0s | チャイム後 |
| 最後のセリフの余韻（締め） | 2.0〜3.0s | ショートの最後 |

## コントの構造と間の置き方

ショートの最小構成:

```
ボケ（状況説明）→ ツッコミ（即答）→ [間] → ボケの反応（落ち）→ [間]
```

- **ボケ → ツッコミの間**: 不要。ツッコミのスピードが鋭さを作る
- **ツッコミ → 落ちの間**: 必須。視聴者がオチを処理する時間
- **落ちの後の間**: 必須。「言葉もない」「やられた」という余韻

## キャラ設定と会話の整合性

さばきゃんはエンジニアリングの知識がない。「触ったから（=リファクタリングあるある）」はNG。知識がある人間が言える言葉になっている。

**制約を厳密に守る**: さばきゃんが言えるのは「コードを知らない人間が素朴に思うこと」のみ。

- NG: 「触ったから。」（あるあるを知っている前提）
- NG: 「そういうものじゃないの？」（コードは触ると壊れるという知識が前提）
- OK: 「なんで触ったの。」（触る必要があったのかという素朴な疑問）
- OK: 「きれいにしたのに。」（言葉をそのまま返す）

## 「主語なしは伝わらない」

ショートは文脈を共有していない視聴者が見る。最初の一言で「誰が何をしたか」が伝わらないと、笑いの前に離脱する。

「コードをきれいにしたら、壊れました。」→ OK（「コードを」があることで状況が伝わる）
「きれいにしようとしたら、動かなくなった。」→ NG（何を？が抜けている）

## creator へ渡すプロンプトの注意点

creator は「簡単さ」を伝えようとすると定型表現に頼りやすい:

- **NG フレーズ**: 「〜だけ」「〜一言」「〜するだけ」「台本1行〜」
- **NG 理由**: 繰り返すとチャンネル全体が安っぽく見える

制約として明示してプロンプトに入れること。

## 教訓

- **ポーズを入れる前に「何のためか」を問う**: 答えられなければ入れない
- **キャラ設定は制約として厳密に守る**: 「さばきゃんはエンジニア知識がない」は会話の自然さに直結する
- **主語を省かない**: ショートは文脈ゼロで見られる。最初の発話で状況が確定すること
- **笑いはセリフではなく間で作る**: セリフを増やすより「無言の落ち＋間」の方が刺さる

# 感情アクション拡張: 同じ方向でも「振れ幅・周波数・累積」でキャラクターを区別する

ep3.5.2 で angry / sad / smile / troubled / chibi の5種を追加したときの設計記録。

ep3.4.4 の設計（joy=縦・surprise=横・thinking=ズーム）は「方向を変えることで干渉を避ける」だったが、
今回は同じ方向の動きが3ペア発生した。それをどう区別したかの記録。

---

## 問題

追加する5種のうち、いくつかは既存の感情と同じ方向に動く。

| 感情 | 方向 | 既存と被る |
| --- | --- | --- |
| angry | 横揺れ | surprise（横）と被る |
| smile | 縦揺れ | joy（縦）と被る |
| troubled | 縦揺れ | joy（縦）と被る |
| chibi | 縦揺れ | joy（縦）と被る |
| sad | 縦（下方向） | 独立 |

「方向で区別する」ep3.4.4 の設計パターンが通用しない。

---

## 解決: 4軸のうち残りの3軸を使う

感情アクションの「動き」を分解すると、4つの軸がある。

| 軸 | 意味 | 例 |
| --- | --- | --- |
| 方向 | 横か縦かズームか | angry=横、joy=縦、thinking=拡大 |
| 振れ幅 | 動きの大きさ（px） | angry=±8、surprise=±3 |
| 周波数 | 動きの速さ（Hz） | chibi=3.0Hz、troubled=2.0Hz |
| 累積 | 時間で変化するか・振動するか | sad=累積、他=振動 |

ep3.4.4 は「方向」だけで3種を区別できた。ep3.5.2 では残り3軸を使った。

---

## 各ペアの設計

### angry vs surprise（同じ横揺れ・振れ幅で区別）

```python
# surprise: ±3px
offset_x = random.choice([-3, 3])

# angry: ±8px（2.5倍以上）
offset_x = random.choice([-8, 8])
```

同じ `random.choice([-amp, amp])` のコードで、`amp` の値だけが違う。
振れ幅が大きいほど「激しい・強い感情」に見える。実装は1行の差。

### smile / troubled / chibi（同じ縦揺れ・2軸で区別）

joy は「idle を置き換える縦揺れ（振幅2倍・周波数2倍）」。
smile / troubled / chibi も縦揺れだが、それぞれ役割が違う。

```
joy     : 振幅×2.0、周波数×2.0、idle を置き換え → 大きく弾む
smile   : 振幅×1.0、周波数×1.0、idle を置き換え → 落ち着いた揺れ（idle と同じリズム）
troubled: 振幅 4px、周波数 2.0Hz、idle に加算   → 小刻みに震える
chibi   : 振幅 5px、周波数 3.0Hz、idle に加算   → 速く小刻みに弾む
```

**smile は「idle と同じ振幅・周波数」で、置き換えによって「idle と同じだが感情が乗っている状態」を表現。**
**troubled / chibi は idle に加算して、それぞれ 2.0Hz / 3.0Hz の速さの差で「おろおろ」と「はしゃぎ」を区別。**

### sad（累積型・他の感情と構造が違う）

```python
# 振動型（他の感情）
offset_y = int(amplitude * math.sin(2 * math.pi * frequency * t))
# → t が変わるたびに sin 波を往復する

# 累積型（sad）
sink = (scene_elapsed / sad_sink_duration_sec) * sad_sink_max_px
total_dy = min(sink + wave, sad_sink_max_px)
# → scene_elapsed が増えるほど下に沈む（最大値でクランプ）
```

sad だけが「時間が経つほど状態が変わる」構造になっている。
喜びや驚きは「その瞬間の感情」だが、悲しみは「時間をかけて積み重なる感情」という性質の違いを表現した。

微小なサイン波（wave）を重畳することで、単調な沈みに「よろめき」を加えている。

## idle との合算ルールの全体像（ep3.5.2 時点）

| 感情 | アイドルとの関係 | 理由 |
| --- | --- | --- |
| joy | 置き換え | 同じ縦方向で加算すると2倍になりすぎる |
| smile | 置き換え | 同上 |
| surprise | 加算（横）| 縦×横で方向が違い干渉しない |
| angry | 加算（横）| 同上 |
| thinking | 加算（ズーム）| 揺れながら拡大しても自然 |
| troubled | 加算（縦・小振幅）| 振幅が小さいため加算しても不自然にならない |
| chibi | 加算（縦・小振幅）| 同上 |
| sad | 加算（縦・下方向）| 沈み量が小さければ加算で自然に見える |

**判断基準**: 「同じ方向で振幅が大きい」ときだけ置き換える。それ以外は加算。

---

## 教訓

**方向が足りなくなったら、振れ幅・周波数・累積の軸を使う。**

ep3.4.4 の時点では「方向を変えれば干渉しない」設計で十分だった。
今回は9種まで増えたため、同じ方向に複数の感情が並んだ。

この場合の設計指針:
- **強さを変えたい**（怒り vs 驚き）→ 振れ幅
- **テンポを変えたい**（おろおろ vs はしゃぎ）→ 周波数
- **時間的な変化を表したい**（悲しみが積み重なる）→ 累積型に切り替える

`calc_emotion_offset()` の `elif` チェーンにブランチを追加するだけで拡張できる設計は、
感情の種類が増えても変わらない。

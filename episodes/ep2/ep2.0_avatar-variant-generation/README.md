# ep2.0 IP-Adapterで表情バリエーションを生成する（失敗実録）

動画「IP-Adapterで1枚のアバターから表情バリエーションを生成する（失敗実録）」のサンプルコードです。

> **注意**: このコードは「うまくいかなかった」実録です。
> なぜうまくいかなかったのか、次に何を試すべきかは動画の失敗分析シーンをご覧ください。

## この動画でやること

IP-Adapter を使って、1枚のアバター画像から5種類の表情バリエーションを生成しようとします。

```
avatar.png（参照）→ IP-Adapter + Stable Diffusion 1.5 → joy / thinking / angry / pointing / nod
```

## セットアップ

```bash
pip install diffusers transformers accelerate torch pillow
```

> GPU がない場合は時間がかかります。CPU でも動きますが、1表情あたり数分〜十数分かかる可能性があります。

## 使い方

```bash
python generate_variants.py avatar.png
# → output/ に 5枚の画像が生成される
```

## 失敗の原因（動画の結論）

IP-Adapter は「キャラクターの外見」を参照する仕組みですが、**表情を変えることに特化したツールではありません**。

| アプローチ | 何をするか | 今回の問題 |
| --- | --- | --- |
| **生成**（IP-Adapter） | 参照画像を元に新しい画像を作る | キャラがブレる |
| **変形** | 既存の画像を起点に表情だけ動かす | 次回試す |

「新しい画像を生成したいのか、既存の画像を変形したいのか」を最初に決めることが重要でした。

# ep4.0 ライフイベント家計シミュレーター — 要件定義

この回では、FP（さばきゃん）の困りごとをゆむが整理し、Claude Code に要件定義を行わせました。

実装コードは別リポジトリで管理しています。

**[→ t-motoki/demo-life-event-simulator](https://github.com/t-motoki/demo-life-event-simulator)**

---

## この回でやったこと

1. さばきゃんの困りごとをヒアリング（毎月 Excel で手計算している家計シミュレーション）
2. ゆむが要求を整理（誰が使う・何を入力・何が出力）
3. Claude Code に要件定義を依頼 → `docs/requirements.md` を生成
4. 生成された要件の末尾に「FP への確認事項」が自動列挙されていた
5. さばきゃんが回答（インフレ率 0% 固定・運用なし・ローン控除追加）→ 要件確定

---

## Claude Code に渡したプロンプト

### 要件定義の依頼（S05）

```
FPが面談で使うライフイベント家計シミュレーターの要件を整理して、
docs/requirements.md に保存してください。

概要：ライフイベント（住宅購入・出産・教育・老後など）のタイミングと収入を入力すると、
年ごとのキャッシュフロー一覧をExcelで出力する。FPが操作する業務ツール。

計算モデルの前提（インフレ率・運用・年金計算方法など）は
FPに確認が必要な事項として別途まとめてください。
```

### FP 回答の反映（S08）

```
FPから確認事項への回答が得られました。requirements.md を更新してください。

- インフレ率: 0%固定（加算しない）
- 運用利回り: 0%固定（考慮しない）
- 住宅ローン控除: 年末ローン残高から計算する（この項目を要件に追加してください）
```
